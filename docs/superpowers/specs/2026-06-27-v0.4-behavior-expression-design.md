# PicoPet v0.4.0 行为表现增强设计规格

## 1. 背景

`v0.3.1` 已作为 Windows 正式版发布，当前产品具备透明置顶桌宠窗口、托盘控制、设置窗口、便携数据目录、安装版和绿色版发布流程。行为系统已经有 `idle / walk / sleep / happy / dragged` 五个状态，以及 `quiet / normal / lively` 三档行为强度、`stationary / short_range` 两种移动模式和可配置睡眠等待时间。

当前不足是各状态视觉差异仍偏弱，主要依赖同一套 idle atlas 和少量缩放、透明度、窗口位移效果。`v0.4.0` 的目标是在不扩大资源体系和配置面的前提下，让桌宠更像“活着”，同时保持轻量、可控、低打扰。

## 2. 目标

- 保留现有五个行为状态，不新增状态。
- 不新增图片、spritesheet、皮肤包或资源加载体系。
- 不新增设置窗口字段，继续使用现有行为配置。
- 通过程序化渲染效果增强 `idle / walk / sleep / happy / dragged`。
- 让 `quiet / normal / lively` 同时影响行为频率和效果幅度。
- 默认 `quiet` 保持保守，升级后不明显打扰用户。
- 保持 Windows-only、轻量、绿色版友好和现有 release gate。

## 3. 非目标

- 不开放 `roaming`。
- 不做新的状态，例如 `curious`、`drowsy`、`startled`、`stretch` 或 `attention`。
- 不兼容 Codex pets 的 9 行动画资源 contract。
- 不做多状态 spritesheet、外部皮肤包、本地皮肤目录或资源导入器。
- 不引入 React、Vue、Svelte、Tailwind、物理引擎、worker、数据库或大型依赖。
- 不新增 AI、提醒、语音、插件、自动更新或跨平台能力。
- 不改安装包、绿色版 `PicoPet/data/` 架构或配置文件位置。

## 4. 用户体验原则

### 4.1 默认保守

默认配置仍是 `quiet`。用户从 `v0.3.1` 升级到 `v0.4.0` 后，不应突然觉得桌宠频繁晃动、弹跳或分散注意力。`quiet` 可以有可感知的细节，但必须低频、小幅、安静。

### 4.2 三档强度含义

`quiet / normal / lively` 同时控制：

- 行为触发频率，例如 idle 微动作和 walk 间隔。
- 单次效果幅度，例如弹跳高度、摆动角度、呼吸幅度、压缩幅度。

三档的产品含义：

- `quiet`：常驻友好，低频微动作，动作幅度小。
- `normal`：默认之外的可见生命感，中等频率和幅度。
- `lively`：更明显、更活跃，但仍有最大幅度限制，不能变成干扰项。

### 4.3 现有设置不扩张

设置窗口继续只暴露现有行为相关字段：

- 行为强度：`quiet / normal / lively`
- 移动模式：`stationary / short_range`
- 睡眠等待时间：`60` 到 `86400` 秒

`v0.4.0` 不增加“增强表现”总开关，也不增加每个效果的细分开关。如果后续用户反馈“太动了”，再在 `v0.4.1` 或后续版本调参数或增加开关。

## 5. 状态表现设计

### 5.1 `idle`

`idle` 是桌宠最常驻的状态，必须保持安静。

- `quiet`：大部分时间保持稳定，低频微动作，例如轻微呼吸、轻微上下浮动、非常小的左右摆动。
- `normal`：持续轻微动效，呼吸和微摆动更容易看见。
- `lively`：更明显的小动作，但限制最大摆动角度和位移，避免桌面干扰。

`idle` 不移动真实窗口，只作用于渲染效果。

### 5.2 `walk`

`walk` 继续由 `behavior.walk_mode` 决定表现路径：

- `stationary`：不移动真实窗口，只做原地走路表现，例如轻微左右摆、上下节奏、帧率略提升。
- `short_range`：沿用当前小范围窗口移动逻辑，同时增强走路摆动、节奏和轻微压缩。

`short_range` 结束时仍回到 anchor position，避免窗口漂移。任何移动失败都应保留当前已有恢复策略。

### 5.3 `sleep`

`sleep` 不新增 `drowsy` 状态，但在进入 `sleep` 后使用 elapsed 时间分阶段表现：

- 前 2 到 3 秒：犯困/低头过渡，表现为轻微下沉、透明度逐步降低、动画速度下降。
- 稳定睡眠循环：慢呼吸、低透明度、低 FPS multiplier。

短按唤醒睡眠时继续回到 `idle`，不触发 `happy`。这保持当前交互语义。

### 5.4 `happy`

`happy` 是点击或短按后的正反馈，按强度分层：

- `quiet`：克制上弹、轻微放大、快速回落。
- `normal`：弹跳加轻微左右摆。
- `lively`：更明显的弹跳和摆动，但限制最大 scale、offset 和 rotation。

`happy` 持续时间仍由 profile 控制。结束后回到 `idle`。

### 5.5 `dragged`

拖拽中使用轻微压缩和下沉表达“被抓住”：

- `scaleX` 略增。
- `scaleY` 略减。
- `offsetY` 略增。
- FPS multiplier 降低。

拖拽结束后不新增状态，但在回到 `idle` 的短时间内表现一次回弹。回弹由前端记录 `lastDragCompletedAt`，在 `idle` 渲染时叠加一个固定时长的 rebound effect；该时间戳不写入配置，不影响状态机语义。`v0.4.0` 不读取拖拽方向或速度，方向/速度反馈留给 `v0.4.2` 鼠标互动阶段。

## 6. 架构设计

### 6.1 数据流

数据流保持现有结构：

```text
PicoPet/data/config.json
  -> Rust get_app_config / picopet://config
  -> frontend BehaviorController
  -> BehaviorProfile
  -> renderEffectForState()
  -> PetRenderer
```

Rust 继续负责配置读取、保存、修复、窗口位置和系统能力。前端继续负责行为状态、渲染效果和动画循环。

### 6.2 `BehaviorController`

`BehaviorController` 继续只负责状态切换和交互语义：

- `idle`
- `walk`
- `sleep`
- `happy`
- `dragged`

它不负责具体旋转角度、缩放比例或透明度。状态机不新增状态，避免扩大测试面和配置兼容风险。

需要调整的行为语义：

- `stationary` 也可以进入 `walk` 状态，但不移动真实窗口。
- `short_range` 进入 `walk` 后继续移动真实窗口。
- `sleep` 仍由 `sleep_after_idle_seconds` 触发。
- pause 或 hidden 时仍回到安静状态，不继续自主行为。

### 6.3 `BehaviorProfile`

`BehaviorProfile` 继续由 `preset` 生成，扩展为更完整的表现参数来源。实现必须提供以下参数，字段名可在实现计划中细化，但语义不能缺失：

- 状态持续时间：happy、walk。
- 自主行为间隔：walk 或 idle 微动作节奏。
- 移动距离：short_range walk distance。
- idle 效果幅度：呼吸、上下浮动、旋转。
- walk 效果幅度：摆动、上下节奏。
- sleep 过渡和稳定呼吸幅度。
- happy 弹跳和摆动幅度。
- dragged 压缩幅度和 idle rebound 幅度。
- FPS multiplier。

参数必须有上限，特别是 `lively`，避免过度弹跳或快速晃动。

### 6.4 `RenderEffect`

`RenderEffect` 扩展以支持更自然的程序化效果：

```ts
export type RenderEffect = {
  scale: number;
  scaleX: number;
  scaleY: number;
  rotationDeg: number;
  offsetX: number;
  offsetY: number;
  alpha: number;
  fpsMultiplier: number;
};
```

默认值：

- `scale = 1`
- `scaleX = 1`
- `scaleY = 1`
- `rotationDeg = 0`
- `offsetX = 0`
- `offsetY = 0`
- `alpha = 1`
- `fpsMultiplier = 1`

最终绘制缩放可理解为：

- X 轴：`scale * scaleX`
- Y 轴：`scale * scaleY`

### 6.5 `PetRenderer`

`PetRenderer` 负责把 `RenderEffect` 应用到 canvas 绘制。

渲染策略：

- 使用 canvas `save()` / `restore()` 包住 transform。
- 以帧中心为旋转和缩放原点。
- 先移动到中心点，再应用 offset、rotation、scale，最后绘制帧。
- 不改变 atlas manifest 格式。
- 不改变 canvas CSS 尺寸计算。

渲染必须保持透明背景，不引入阴影、光效或额外 DOM。

### 6.6 `app.ts`

`app.ts` 继续负责组合行为状态、渲染效果、动画帧和窗口移动。

需要明确两条 walk 路径：

- 当 `snapshot.state === "walk"` 且 `walk_mode === "short_range"` 时，调用 `moveWindowTo()`。
- 当 `snapshot.state === "walk"` 且 `walk_mode === "stationary"` 时，不调用 `moveWindowTo()`，只渲染走路效果。

`returnWindowToAnchor()` 只处理真实移动路径，不能让 `stationary` 产生位置副作用。

## 7. 错误处理和回归边界

- 图片加载失败时继续使用现有 fallback。
- canvas transform 失败不应影响主流程；实现中不应抛出可见运行时异常。
- `moveWindowTo()` 失败时沿用现有 walk 失败恢复策略。
- 配置中未知 preset 或 walk mode 继续由 Rust sanitize 修复。
- `behavior.enabled = false` 时不执行自主行为增强。
- `animation.paused = true` 或 `document.hidden = true` 时不继续主动调度动画。
- 点击穿透开启时继续不响应 canvas pointerdown。

## 8. 测试策略

### 8.1 TypeScript 单元测试

必须覆盖：

- `RenderEffect` 默认值包含 `scaleX / scaleY / rotationDeg`。
- `renderEffectForState("idle", ...)` 在三档强度下产生不同频率或幅度。
- `renderEffectForState("sleep", ...)` 前 2 到 3 秒为过渡效果，稳定后为低频呼吸效果。
- `renderEffectForState("happy", ...)` 三档强度分层，并限制最大 scale、offset 和 rotation。
- `renderEffectForState("dragged", ...)` 产生压缩和下沉效果。
- `walk` 渲染效果包含原地走路表现。
- `BehaviorController` 在 `stationary` 下仍可进入 `walk`，供前端渲染原地走路。
- `PetRenderer` 使用 `scaleX / scaleY / rotationDeg` 绘制时不会破坏基本绘制路径。

### 8.2 Rust 测试

Rust 配置结构不新增字段，因此不需要新增配置迁移测试。保留现有测试：

- preset sanitize。
- walk mode sanitize。
- window position。
- settings commands。
- release subsystem。

如果实现中调整 Rust 暴露类型或命令返回结构，需要补充对应测试。

### 8.3 手动 QA

新增 `docs/qa/windows-v0.4-behavior-expression-checklist.md`，覆盖：

- 默认 `quiet` 是否安静。
- `normal` 和 `lively` 是否可见增强。
- `idle` 是否不频繁打扰。
- `stationary` walk 是否不移动窗口。
- `short_range` walk 是否仍能小范围移动并回到 anchor。
- `sleep` 是否有过渡和稳定睡眠表现。
- 点击后 `happy` 是否自然。
- 拖拽中压缩、松手回弹是否自然。
- 暂停动画、点击穿透、设置窗口、退出/重启位置恢复是否不回归。
- release build 内存仍满足当前 50-100 MB Private Working Set 目标。

## 9. 发布策略

`v0.4.0` 直接作为正式版发布，不先发 beta 或 rc。

发布前必须通过：

- `pnpm check`
- `pnpm release:windows:gate`
- `pnpm artifact:windows`
- Windows 手动 QA checklist
- 内存基线记录

如果发布后体感不舒服，后续通过 `v0.4.1` 调整参数，不在 `v0.4.0` 增加复杂设置开关。

## 10. 验收标准

- 应用仍只有五个行为状态：`idle / walk / sleep / happy / dragged`。
- 不新增配置字段，旧配置可直接升级。
- 不新增图片资源、皮肤系统或新依赖。
- `quiet` 默认保守，`normal / lively` 有可感知差异。
- `stationary` walk 不移动真实窗口。
- `short_range` walk 继续小范围移动真实窗口并返回 anchor。
- `sleep` 有进入过渡和稳定睡眠循环。
- `happy` 和 `dragged` 的反馈更明确但不夸张。
- 暂停、隐藏、点击穿透、拖拽位置保存、重启位置恢复不回归。
- 自动测试覆盖行为和效果函数。
- Windows release gate 通过。

## 11. 后续规划

- `v0.4.1`：根据实际体感调整参数，必要时增加一个总开关。
- `v0.4.2`：更细鼠标互动，例如双击、鼠标靠近、拖拽方向/速度反馈。
- `v0.4.x` 后续：独立评估 `roaming`，不和表现增强混在同一版本。
- `v0.5.x`：再考虑多状态 spritesheet、内置资源结构和本地皮肤目录。

## 12. 决策记录

- 选择保守增强五状态方案，而不是新增状态或提前资源体系。
- 选择程序化轻量效果，而不是新增图片资源。
- 选择三档强度同时影响频率和幅度。
- 选择不新增设置项。
- 选择 `stationary` 原地表现、`short_range` 真实小范围移动。
- 选择 `sleep` 内部过渡，不新增 `drowsy`。
- 选择 `happy` 按三档分层。
- 选择 `dragged` 压缩和松手回弹，不读取拖拽速度或方向。
- 选择扩展 `RenderEffect` 支持 `scaleX / scaleY / rotationDeg`。
- 选择自动测试优先加手动视觉 QA。
- 选择直接发布正式版 `v0.4.0`，不先发 beta。
