# PicoPet v0.2.1 Behavior Config Design

## 1. 背景

`v0.2.0` 已完成并发布稳定版：桌宠已有 `idle / walk / sleep / happy / dragged` 行为状态机，默认 `quiet` 档位，默认 `short_range` 行走模式，配置保存在绿色版目录 `PicoPet/data/config.json`。当前 TypeScript 类型已经预留 `quiet / normal / lively` 和 `stationary / short_range / roaming`，但 Rust 配置修复仍会把未知值修回 `quiet` 与 `short_range`，运行时也只按 quiet + short_range 执行。

`v0.2.1` 的目标是把已经规划的轻量行为参数真正开放一部分，让用户无需等待设置窗口，也能通过配置文件调整桌宠活跃程度。

## 2. 目标

- 启用 `behavior.preset = quiet / normal / lively` 三档行为节奏。
- 启用 `behavior.walk_mode = stationary / short_range` 两种行走模式。
- 保留 `roaming` 作为后续规划值，但 `v0.2.1` 不实现屏幕范围自主漫游。
- 保持 Windows-only，不引入跨平台层改动。
- 保持轻量，不引入 React/Vue/Svelte/Tailwind、物理引擎、插件或 AI。
- 继续通过 `PicoPet/data/config.json` 配置，不做设置窗口。
- 不新增大体积图片资源，不引入每状态独立 spritesheet。
- 保持 `v0.2.0` 的 release gate、绿色版数据目录、位置保存、点击穿透、托盘行为不退化。

## 3. 非目标

- 不做 `v0.3` 设置窗口。
- 不做 `roaming` 全屏/多屏自主漫游。
- 不做每状态独立 atlas，例如 `pico_walk`、`pico_sleep`、`pico_happy`。
- 不做外部皮肤包、资源导入器、皮肤编辑器或插件系统。
- 不做 AI 聊天、提醒、日程、语音或大模型集成。
- 不做安装器交互流程重设计。

## 4. 配置范围

`AppConfig.behavior` 保持现有字段，不新增配置段：

```json
{
  "behavior": {
    "enabled": true,
    "preset": "quiet",
    "walk_mode": "short_range",
    "sleep_after_idle_seconds": 900
  }
}
```

字段规则：

| 字段 | 类型 | 默认值 | `v0.2.1` 规则 |
| --- | --- | --- | --- |
| `enabled` | boolean | `true` | `false` 时禁用自主 `walk` 和 `sleep`，短按不触发 `happy`，接近 `v0.1.0` idle 行为 |
| `preset` | string | `quiet` | 接受 `quiet / normal / lively`，未知值修复为 `quiet` |
| `walk_mode` | string | `short_range` | 接受 `stationary / short_range`，未知值包括 `roaming` 修复为 `short_range` |
| `sleep_after_idle_seconds` | number | `900` | 继续 clamp 到 `60-86400` 秒 |

`roaming` 的处理必须明确：用户写入 `"roaming"` 时，`v0.2.1` 会修复为 `"short_range"`，避免配置文件看起来支持但运行时没有真实实现。

## 5. 行为档位

档位只影响行为频率和反馈强度，不改变状态集。

| Preset | 定位 | happy 持续 | walk 间隔 | walk 持续 | walk 距离 | 渲染效果 |
| --- | --- | ---: | ---: | ---: | ---: | --- |
| `quiet` | 低打扰默认档 | 1400 ms | 240000 ms | 6000 ms | 40 px | 维持 v0.2.0 观感 |
| `normal` | 更明显但仍克制 | 1600 ms | 150000 ms | 7000 ms | 56 px | happy bounce 略强，walk fps 略高 |
| `lively` | 更活跃的桌宠感 | 1800 ms | 90000 ms | 8000 ms | 72 px | happy bounce 更明显，walk fps 更高 |

档位设计原则：

- `quiet` 必须保持 `v0.2.0` 的默认体验，避免稳定版升级后突然变吵。
- `normal` 是后续设置窗口里的推荐用户可见选项。
- `lively` 允许更频繁反馈，但仍不能像游戏角色一样持续移动。
- 所有档位仍遵守 `animation.paused`、窗口隐藏、`behavior.enabled=false` 的禁用规则。

## 6. 行走模式

### 6.1 `stationary`

`stationary` 表示不做自主 walk：

- 不进入 `walk` 状态。
- 不移动窗口。
- 仍允许 `happy`、`sleep`、`dragged`。
- 适合用户只想要点击反馈和睡眠表现，但不希望桌宠自己移动的场景。

### 6.2 `short_range`

`short_range` 保持 v0.2.0 anchor 模型：

- 自主 walk 只在用户保存位置附近移动。
- walk 不保存位置。
- walk 完成后回到 anchor。
- walk 中发生拖拽时取消 walk，并以用户拖拽结束位置作为新的 anchor。
- 距离和持续时间由 `preset` 决定。

### 6.3 `roaming`

`roaming` 不在 `v0.2.1` 实现：

- Rust 配置修复不接受 `roaming`。
- TypeScript 类型可以继续保留 `roaming` 作为未来值，但运行时不得产生 roaming 行为。
- 文档中说明 roaming 是后续版本规划，不作为当前配置项宣传。

## 7. 架构设计

### 7.1 Rust 配置修复

`src-tauri/src/config.rs` 继续拥有配置修复权：

- `BehaviorConfig.preset` 接受 `quiet / normal / lively`。
- `BehaviorConfig.walk_mode` 接受 `stationary / short_range`。
- 其他值修复为默认值。
- 默认值保持 `quiet` + `short_range`。
- 旧配置加载后继续补齐 `behavior` 段。

Rust 侧不新增 Tauri command，不新增数据迁移文件。配置修复仍由 `ConfigStore::load_or_repair()` 写回 `config.json`。

### 7.2 TypeScript 行为模块

现有 `src/pet/behavior/` 继续作为行为边界：

```text
src/pet/behavior/
  types.ts
  controller.ts
  motion.ts
  effects.ts
  timing.ts
```

职责调整：

- `timing.ts` 从 `createQuietBehaviorTiming()` 扩展为按 preset 返回 timing profile。
- `motion.ts` 支持从 preset timing/profile 获取 distance，不把距离硬编码在 `app.ts`。
- `controller.ts` 根据 `walk_mode` 判断是否允许进入 `walk`。
- `effects.ts` 根据 `preset` 调整 `happy` 和 `walk` 的视觉强度。
- `types.ts` 保持 `BehaviorPreset` 和 `WalkMode` 明确类型。

`src/app.ts` 只做接线，不拥有档位常量。它应从 controller/effects/motion 输出中读取当前状态和 profile。

## 8. 状态规则

状态优先级保持不变：

```text
dragged > happy > sleep > walk > idle
```

规则补充：

- `walk_mode=stationary` 时，`idle` 不会自主进入 `walk`。
- `preset` 只影响 timing/effect，不绕过状态优先级。
- `sleep_after_idle_seconds` 优先级高于 walk：达到睡眠时间时先进入 `sleep`。
- `sleep` 中点击只唤醒，不叠加强烈 happy。
- `animation.paused=true` 时不进入 `walk`、`sleep`、`happy`。
- `behavior.enabled=false` 时不进入 `walk`、`sleep`、`happy`。

## 9. 低功耗与内存约束

- 不新增常驻 interval。
- 继续复用 `createAnimationLoop()` 和 requestAnimationFrame。
- `stationary` 不应产生窗口移动调用。
- `quiet` 默认体验的内存目标继续是 50-100 MB Private Working Set。
- `normal/lively` 只改变既有 loop 的状态参数，不创建额外 WebView、worker 或大型资源。
- release gate 必须继续生成 NSIS 和 portable zip。

## 10. 错误处理

- 未知 `preset`：修复为 `quiet` 并写回配置。
- 未知 `walk_mode`：修复为 `short_range` 并写回配置。
- `sleep_after_idle_seconds` 超出范围：继续 clamp 到 `60-86400`。
- 窗口移动失败：取消当前 walk，回到 `idle`，不保存自主移动位置。
- click-through 开启时收不到点击：保持现有行为，视为预期限制。
- 配置文件损坏：沿用当前 damaged config repair，恢复默认配置。

## 11. 测试计划

### 11.1 Rust 单元测试

- 默认配置仍为 `quiet` + `short_range`。
- `preset=normal` 保留并写回。
- `preset=lively` 保留并写回。
- 未知 `preset` 修复为 `quiet`。
- `walk_mode=stationary` 保留并写回。
- `walk_mode=short_range` 保留并写回。
- `walk_mode=roaming` 修复为 `short_range`。
- 旧配置仍补齐默认 behavior。

### 11.2 TypeScript 单元测试

- `timing.test.ts`
  - `quiet / normal / lively` 返回不同 timing。
  - 未知 preset 通过配置修复后不应进入前端；前端仍对非预期值 fallback 到 quiet。
- `controller.test.ts`
  - `walk_mode=stationary` 不进入 walk。
  - `walk_mode=short_range` 在到达 nextWalkAt 后进入 walk。
  - `preset` 改变 walk/happy 时长。
  - `enabled=false` 禁止自主行为。
- `motion.test.ts`
  - short-range walk 距离来自 profile。
  - walk 完成后仍回到 anchor。
- `effects.test.ts`
  - `quiet / normal / lively` 的 happy 效果强度有序增强。
  - `stationary` 不改变 render effect；它只阻止 walk 状态。

### 11.3 集成与发布验证

- `pnpm test -- --run`。
- `pnpm build`。
- `cargo test --manifest-path src-tauri/Cargo.toml`。
- `pnpm release:windows:gate`。
- 手动检查 `PicoPet/data/config.json`：
  - 设置 `preset=normal` 后重启，点击反馈与 walk 频率比 quiet 更明显。
  - 设置 `preset=lively` 后重启，反馈更活跃但不会持续乱动。
  - 设置 `walk_mode=stationary` 后重启，观察期间不自主移动。
  - 设置 `walk_mode=roaming` 后重启，配置被修复为 `short_range`。

## 12. 文档更新

- README 的 behavior 配置示例保持默认 `quiet` + `short_range`。
- README 增加可选值说明：
  - `preset`: `quiet / normal / lively`
  - `walk_mode`: `stationary / short_range`
  - `roaming` planned, not available in `v0.2.1`
- 新增 `docs/qa/windows-v0.2.1-checklist.md`，记录配置手动验证项。
- `docs/release/windows-release-process.md` 不需要改变流程，继续使用 release gate。

## 13. 验收标准

- 默认配置和默认体验与 `v0.2.0` 接近，不让已有用户升级后明显被打扰。
- `quiet / normal / lively` 在单元测试中有可验证差异。
- `stationary` 能可靠阻止自主 walk，但不影响点击、睡眠、拖拽。
- `short_range` 继续保护用户保存位置，walk 后回到 anchor。
- `roaming` 不被误认为已支持。
- 损坏配置、旧配置、未知配置值均能被修复。
- `pnpm release:windows:gate` 通过。

## 14. 风险与应对

| 风险 | 应对 |
| --- | --- |
| `lively` 太频繁影响常驻体验 | 默认仍为 `quiet`，文档标注 lively 适合主动体验 |
| 配置值开放后测试复杂度上升 | 用 profile 表集中定义 preset，controller 只消费 profile |
| `roaming` 类型存在导致误用 | Rust sanitize 不接受 roaming，文档明确 planned only |
| app.ts 继续堆积逻辑 | 档位常量放在 behavior 模块，app.ts 只接线 |
| 自主移动影响位置保存 | 保持 anchor 规则，walk 不调用保存位置 |

## 15. 规格自查

- 本规格只覆盖 `v0.2.1` 的行为配置增强，不包含设置窗口。
- 本规格明确启用 `quiet / normal / lively`。
- 本规格明确启用 `stationary / short_range`，不实现 `roaming`。
- 本规格不引入新依赖、不引入大素材、不改变 Windows-only 范围。
- 本规格保留 `v0.2.0` 默认体验和 release gate。
- 本规格给出 Rust、TypeScript、文档和 QA 的验收标准。
