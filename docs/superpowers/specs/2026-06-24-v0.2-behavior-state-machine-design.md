# PicoPet v0.2 Behavior State Machine Design

## 1. 背景

`v0.1.0` 已完成 Windows 稳定版收尾：透明置顶桌宠窗口、内置 idle 动画、托盘控制、点击穿透、拖拽位置保存、绿色版 `PicoPet/data/`、安装器 QA 和 release gate 均已发布验证。`v0.2.x` 的优先方向是桌宠交互和行为增强。

当前前端只有单一 idle atlas 播放逻辑：`src/app.ts` 负责加载 `pico_idle` 资源、启动 animation loop、处理拖拽保存；`src/pet/animationClock.ts` 负责帧序号计算；`src/pet/renderer.ts` 只支持渲染 atlas 帧和 fallback。Rust 侧已有 `AppConfig.animation.idle_fps`、`interactive_fps`、`paused`，但没有行为状态配置。

## 2. 目标

- 增加轻量行为状态机，让桌宠在 `idle / walk / sleep / happy / dragged` 之间切换。
- `v0.2.0` 不新增大量图片资源，复用现有 idle atlas，并通过程序化表现增强行为感。
- 保持轻量、低打扰、低内存，不引入 React/Vue/Svelte/Tailwind，不引入物理引擎。
- 保持 Windows-only，不做 macOS/Linux 平台层。
- 保持绿色版数据模型，新增行为配置写入 `PicoPet/data/config.json`。
- 保留未来迁移到每状态独立 spritesheet 的接口边界。

## 3. 非目标

- 不做设置窗口；设置窗口留到 `v0.3`。
- 不做外部皮肤包、资源导入器、皮肤编辑器或插件系统。
- 不做 AI 聊天、提醒、日程、语音或大模型集成。
- 不做屏幕范围自主漫游；`roaming` 只进入规划，不在 `v0.2.0` 实现。
- 不做每状态独立 spritesheet；该能力进入 `v0.2.x` 后续完善规划。

## 4. 版本范围

### 4.1 `v0.2.0` 实现

- 基础状态集：`idle / walk / sleep / happy / dragged`。
- 默认行为档位：`quiet`。
- 默认走路模式：`short_range`。
- 点击触发轻量 `happy` 反馈，持续 1-2 秒后回到 `idle`。
- 长时间无互动后进入 `sleep`，默认 900 秒，可通过配置文件修改。
- 拖拽期间进入 `dragged`，暂停自主行为，拖拽结束后保存位置并回到 `idle`。
- `walk` 只在用户保存位置附近做小范围移动，完成后回到锚点，不永久修改用户保存位置。

### 4.2 `v0.2.x` 后续规划

- `walk_mode` 扩展为 `stationary / short_range / roaming` 设置项。
- `behavior.preset` 扩展为 `quiet / normal / lively` 三档设置项。
- 每个行为状态支持独立 atlas：`pico_idle`、`pico_walk`、`pico_sleep`、`pico_happy`、`pico_dragged`。
- 稳定后将行为参数暴露给 `v0.3` 设置窗口。

## 5. 行为状态

| State | 触发 | 表现 | 退出 |
| --- | --- | --- | --- |
| `idle` | 启动、其他状态结束、唤醒 | 正常 idle atlas，低频自主计时 | 触发 `walk`、`sleep`、`happy`、`dragged` |
| `walk` | `quiet` 档下偶尔触发 | 窗口在锚点附近短距离左右移动，动画使用 idle atlas | 走动结束后回到锚点并进入 `idle` |
| `sleep` | 无点击/拖拽超过 `sleep_after_idle_seconds` | 慢呼吸、低频帧率或轻微透明/缩放表现 | 点击或拖拽唤醒到 `idle` 或 `dragged` |
| `happy` | 短按桌宠且移动未超过拖拽阈值 | 短暂 bounce、轻微放大、动画节奏加快 | 1-2 秒后回到 `idle` |
| `dragged` | 左键 pointer down 且未开启 click-through | 暂停自主状态，表现为 held 状态 | 短按释放进入 `happy`，拖拽释放保存位置后进入 `idle` |

状态优先级固定为：

```text
dragged > happy > sleep > walk > idle
```

当 `animation.paused = true` 时，不触发自主 `walk` 和 `sleep`，也不启动 `happy` 反馈；已经处于 `dragged` 时仍允许完成拖拽保存。

## 6. 配置设计

`AppConfig` 新增 `behavior` 段：

```json
{
  "behavior": {
    "enabled": true,
    "preset": "quiet",
    "walk_mode": "short_range",
    "sleep_after_idle_seconds": 900
  }
}
```

字段规则：

| 字段 | 类型 | 默认值 | `v0.2.0` 规则 |
| --- | --- | --- | --- |
| `enabled` | boolean | `true` | `false` 时状态机不做自主切换，只保留当前 `v0.1.0` 行为 |
| `preset` | string | `quiet` | 只实现 `quiet`；未知值修复为 `quiet` |
| `walk_mode` | string | `short_range` | 只实现 `short_range`；未知值修复为 `short_range` |
| `sleep_after_idle_seconds` | number | `900` | 小于 60 修复为 60，大于 86400 修复为 86400 |

未来规划值：

```text
preset: quiet / normal / lively
walk_mode: stationary / short_range / roaming
```

配置修复仍由 Rust `ConfigStore::load_or_repair()` 负责。损坏配置和旧版本配置加载后必须补齐 `behavior` 段并写回 `config.json`。

## 7. 架构

### 7.1 TypeScript 前端

新增行为模块，放在 `src/pet/behavior/`：

```text
src/pet/behavior/
  types.ts
  controller.ts
  motion.ts
  effects.ts
  timing.ts
```

职责：

- `types.ts`：定义 `BehaviorState`、`BehaviorConfig`、`BehaviorEvent`、`AnimationProfile`、`MotionProfile`。
- `controller.ts`：纯状态机，不直接访问 DOM、Tauri 或 canvas。
- `motion.ts`：计算 short-range walk 的窗口偏移，不直接持久化配置。
- `effects.ts`：根据状态输出渲染变换，例如 scale、offset、alpha、frame-rate multiplier。
- `timing.ts`：提供可测试的时间参数和 quiet 档调度规则。

`src/app.ts` 只负责接线：

- 加载配置。
- 创建 `BehaviorController`。
- 将 pointer/click/drag/config/visibility 事件转成状态机事件。
- 将状态机输出传给 renderer 和窗口移动 API。
- 保持原有拖拽保存路径。

### 7.2 Rust 后端

Rust 只新增配置结构和校验：

- `src-tauri/src/config.rs` 新增 `BehaviorConfig`。
- `AppConfig::default()` 包含默认 behavior。
- `AppConfig::sanitized()` 修复未知档位、未知 walk 模式和超出范围的 sleep 秒数。
- Tauri command 不需要新增 API；`get_app_config` 继续返回完整配置。

`v0.2.0` 不改 Windows 平台层，不改安装器数据目录策略，不改 release artifact 结构。

## 8. 走路行为边界

`short_range` walk 必须保护用户保存位置：

- 用户拖拽保存的位置称为 anchor。
- 自主 walk 只在 anchor 附近短距离移动窗口。
- walk 结束时窗口回到 anchor。
- walk 过程中不调用 `saveWindowPosition()`。
- 如果用户在 walk 过程中拖拽，立即进入 `dragged`，取消 walk，并以用户拖拽结束位置作为新的 anchor。

默认 quiet 参数：

```text
walk_interval_seconds: 180-300
walk_duration_seconds: 4-8
walk_distance_px: 24-64
```

`v0.2.0` 可以先用固定 quiet 参数；随机性必须注入或封装，保证单元测试可预测。

## 9. 点击与拖拽事件

点击行为：

- pointer down 后进入 held/`dragged` 表现。
- 如果 pointer up 前移动未超过拖拽阈值，则按短按处理并触发 `happy`。
- 短按必须重置 sleep 计时。
- `happy` 持续 1-2 秒。
- `sleep` 状态下单击优先唤醒，不叠加强烈反馈。

拖拽行为：

- pointer down 后进入 `dragged`。
- 如果 pointer move 超过拖拽阈值，则按拖拽处理。
- `dragged` 期间取消自主 walk 和 sleep 计时。
- 拖拽结束后沿用现有 `persistPositionAfterDrag()` 保存位置。
- 保存完成后进入 `idle` 并重置 anchor。

## 10. 渲染表现

`v0.2.0` 继续使用 `pico_idle.json/png`，状态表现通过程序化效果完成：

| State | Atlas | 程序化效果 |
| --- | --- | --- |
| `idle` | `pico_idle` | 无额外效果 |
| `walk` | `pico_idle` | 水平窗口移动，帧率可略高 |
| `sleep` | `pico_idle` | 慢呼吸缩放，帧率降低 |
| `happy` | `pico_idle` | bounce、轻微放大、短时加速 |
| `dragged` | `pico_idle` | held 姿态效果，暂停自主移动 |

Renderer 需要支持可选 transform 输入，但不负责状态切换：

```ts
type RenderEffect = {
  scale: number;
  offsetX: number;
  offsetY: number;
  alpha: number;
};
```

后续每状态 spritesheet 迁移时，状态机输出从 `RenderEffect` 扩展为 `AnimationProfile`：

```ts
type AnimationProfile = {
  atlasId: "idle" | "walk" | "sleep" | "happy" | "dragged";
  fps: number;
  loop: boolean;
};
```

## 11. 低功耗策略

- 继续复用现有 `createAnimationLoop()`，不要引入额外常驻 interval。
- 页面隐藏时不渲染动画，不触发自主 walk。
- `sleep` 状态降低有效渲染频率。
- `animation.paused` 时停止自主状态调度。
- `walk` 不做逐毫秒物理模拟，只按 requestAnimationFrame 插值短距离位置。

## 12. 错误处理

- 行为配置缺失：修复为默认 behavior。
- 行为配置损坏：沿用当前 damaged config repair 策略，恢复默认配置。
- 窗口移动失败：取消当前 walk，回到 `idle`，不影响用户拖拽保存。
- 图片加载失败：沿用 renderer fallback，状态机仍可运行但不要求可见效果。
- click-through 开启时：前端可能无法收到点击事件，这是预期行为。

## 13. 测试计划

### 13.1 TypeScript 单元测试

- `controller.test.ts`
  - 默认从 `idle` 启动。
  - 单击进入 `happy`，超时回 `idle`。
  - 无互动超过 sleep 时间进入 `sleep`。
  - sleep 中点击唤醒。
  - pointer down 进入 `dragged`，pointer up 回 `idle`。
  - `animation.paused` 禁止自主 `walk` 和 `sleep`。
- `motion.test.ts`
  - short-range walk 不超过配置距离。
  - walk 完成后回到 anchor。
  - walk 取消时不返回旧状态错误。
- `effects.test.ts`
  - 每个状态输出稳定的 render effect。
  - effect 不改变 canvas 固定尺寸。

### 13.2 Rust 单元测试

- 默认配置包含 `behavior`。
- 旧配置加载后补齐 `behavior`。
- 未知 `preset` 修复为 `quiet`。
- 未知 `walk_mode` 修复为 `short_range`。
- `sleep_after_idle_seconds` clamp 到 60-86400。

### 13.3 集成与发布验证

- `pnpm test -- --run`。
- `cargo test --manifest-path src-tauri/Cargo.toml`。
- `pnpm release:windows:gate`。
- 手动验证：
  - 启动后大多数时间保持 idle。
  - 单击触发 happy。
  - 拖拽期间不自主移动。
  - 绿色版 `data/config.json` 出现 behavior 配置。
  - 关闭重启后用户拖拽位置仍恢复。

## 14. 验收标准

- `v0.2.0` 不破坏 `v0.1.0` 的启动、托盘、点击穿透、拖拽保存、绿色版数据目录和 release gate。
- 行为状态机在单元测试中可确定、可复现。
- 默认 quiet 行为低打扰：短时间观察不会频繁移动。
- 自主 walk 不永久修改用户保存位置。
- `sleep_after_idle_seconds` 可通过 `data/config.json` 修改并生效。
- `behavior.enabled = false` 能恢复接近 `v0.1.0` 的静态 idle 行为。

## 15. 风险与应对

| 风险 | 应对 |
| --- | --- |
| 自主 walk 破坏用户保存位置 | 使用 anchor 模型，walk 结束回 anchor，不保存自主移动位置 |
| 状态机和渲染耦合过深 | 状态机保持纯 TypeScript，无 DOM/Tauri 依赖 |
| 随机调度导致测试不稳定 | 随机源封装注入，测试使用固定序列 |
| 行为太活跃影响常驻体验 | 默认 quiet，walk 间隔长，sleep 降频 |
| 后续 spritesheet 迁移困难 | `AnimationProfile` 和 `atlasId` 在设计中预留 |

## 16. 规格自查

- 本规格覆盖 `idle / walk / sleep / happy / dragged` 基础状态集。
- 本规格明确 `v0.2.0` 使用程序化表现，后续 `v0.2.x` 迁移到每状态 spritesheet。
- 本规格明确 `short_range` 为默认 walk 模式，并保留 `stationary / roaming` 规划。
- 本规格明确 `quiet` 为默认行为档，并保留 `normal / lively` 规划。
- 本规格明确 `sleep_after_idle_seconds` 可配置，默认 900 秒。
- 本规格明确设置窗口不属于 `v0.2.0`，留到 `v0.3`。
